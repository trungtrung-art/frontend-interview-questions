## Usage Guide

### For Candidates:

1. **Assess your current level:**
   - Start with Junior section first
   - If correct > 80%, proceed to Middle
   - If correct > 70% Middle, proceed to Senior

2. **Effective learning approach:**
   ```
   1. Read question -> Self-answer
   2. Check answer -> Compare
   3. Read explanation -> Understand why
   4. Follow reference link -> Deep dive
   5. Try coding -> Practice
   ```

3. **Focus by position:**
   - **Junior:** JS basics, React basics, HTML/CSS
   - **Middle:** TypeScript, Testing, Performance basics
   - **Senior:** System Design, Architecture, Leadership

### For Interviewers:

1. Select questions according to position level
2. Mix between categories
3. Follow up with "why" and "how"
4. Evaluate explanation approach, not just correct answers

## Senior Interview Tips

### Technical:
1. **Know not just "what" but also "why" and "when"**
2. **Trade-offs:** Every decision has pros/cons
3. **Experience-based:** Use real examples from projects
4. **System thinking:** See the problem holistically

### Behavioral:
1. **STAR method:** Situation, Task, Action, Result
2. **Ownership:** Show leadership even without title
3. **Growth mindset:** Learn from failures
4. **Communication:** Explain complex concepts simply

### Questions to Ask:
- "What does success look like for this role?"
- "What are the biggest technical challenges?"
- "How does the team handle technical debt?"
- "What's the code review process?"
